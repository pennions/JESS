Layout elements

The elements <main>, <section> and <fieldset> are all displayed with display:flex.
The child elements are all displayed by default at width: 100% which results in a single column.
All columns will be automatically scaled down and wrapped when the viewport can not fit all the columns properly.
You can try it by resizing the browser.

N.B. The blue fill texture is for demonstration purpose only.

Single column

All child elements start out as width:100%; and will take up more space if there is room, up to their specified maximum.

<section>
    <div>Block 100% width</div>
</section>

Center

Add class="center" to get a single element centered vertically and horizontally. It has height:100vh; by default.
It works well for a login screen or a single card that needs to be centered.

It has the following variants available:

  • class="center"
    • It will take 100% and will be centered horizontally.
    • if the screen is has min-width:576px; it will be centered vertically.
  • class="center with-header"
    • It will subtract the height of the header from the area.
  • class="center with-footer"
    • It will subtract the height of the footer from the area.
  • class="center with-header with-footer"
    • It will subtract both height of the header and the footer from the area.
  • content-top
    • If you add content-top class to any of the above, the child will only be horizontally centered and not vertically
N.B. if you want the footer to stick to the bottom, apply a column class on the body and column-end class on the footer. 
<nav>Navigation menu</nav>
<main class="center with-header with-footer">
    <article>
       A centered element with a header and a footer
    </article>
<main>
<footer>Footer area</footer>

Aside layout

The aside layout can be used if you want to have a small - wide layout. horizontal layout.
The aside element has a width:auto;
The element on the right will have flex:1; and take the remaining space.
Only the first element must be <aside>, the right side can be any element. 

<main>
    <aside>
        Element with auto width
    </aside>
    <section>
        Element that takes the remaining space
    </section>
</main>
Fun fact: the current page uses this aside layout for having a vertical navigation bar.

Nav

When you include a <nav> element directly inside a <body> element it will:

  • Have a box-shadow
  • Have max-height: 6rem;
  • Have padding: 1.5rem;

Utility classes

Class Description
sticky Makes it stick to the top when scrolling down.
no-scroll Remove all scrollbars, handy to apply on the body when opening a modal.
full-width Make an element take 100% of its parent size.
overflow-horizontal Make an element crollable on the x-axis if the contents is too large.
no-stretch Apply this to an element inside a flex layout to not make them take up all the space.
d-inline-block Make an element display as an inline-block.
row Make an element display flex with the default row direction.
column Make an element display flex with the column direction
row-end When applied to an element inside a display flex element, you can have it move all the way to the right.
column-end When applied to an element inside a display flex element with the column direction, you can have it move all the way down to the bottom.
center-content-vertical This class can be used combined with row or column class to align all children along the Y-axis (top to bottom).
center-content-horizontal This class can be used combined with row or column class to align all children along the X-axis (left to right).
space-between This class can be used combined with row or column to align have the first and last child on the start and end and have even spaces in between.
auto-width Makes an element take on an auto-width.
no-wrap Makes a text not wrap.
